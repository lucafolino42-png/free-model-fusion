💾 Session Memory

What this is: Per-session conversation history. The Chat API takes a sessionId; messages you send on the same id are remembered and fed back to the models as prior turns. Sessions also persist your chosen routing profile, web-search mode, and custom model set.

Session ID conventions: The dashboard defaults to default. Telegram chats use telegram:<chat-id>. Webhooks use whatever you pass in the body. Pick a unique id per context (e.g. work-research, client-acme) to keep streams separate.

Clearing memory deletes only messages; the session row, profile, and preferredExperts are kept so reloading shows the same profile again.
